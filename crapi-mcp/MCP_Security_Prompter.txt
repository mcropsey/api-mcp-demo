  SLIDE 1  —  The Stats Tell a Different Story
  Target: ~2 min
═══════════════════════════════════════════════════

Eighty-seven percent of organizations suffered an
API security incident last year.

That's a four-year high.

Forty-two percent trace those incidents directly to
APIs powering AI applications and agents.

And only 23% — fewer than one in four — actually
know which of their APIs return sensitive data.

These are not AI numbers. These are API numbers.

The 42% isn't a warning about the model —
it's a warning about what the model calls underneath.

Everyone in this room is focused on AI security
right now. That concern is completely valid.

But here's the problem:

Every AI security concern becomes irrelevant if the
API foundation that AI depends on is already broken.

The vulnerabilities I'm about to show you exist
today — in production — with or without AI.

Broken authorization.
Overexposed data.
Shadow endpoints nobody is watching.

AI didn't create any of them.
AI just finds them in seconds instead of hours.

You can't secure the supercar if the roads
underneath it are full of potholes.

Fix the foundation first.

  SLIDE 2 / DEMO SETUP  —  30 seconds
═══════════════════════════════════════════════════

I'm connected to crAPI — Completely Ridiculous API —
a deliberately vulnerable application used for
security training.

Claude is connected to it through a custom MCP server.

MCP — Model Context Protocol — is the standard that
lets AI agents call external services directly.

No browser. No Postman. Just natural language.


───────────────────────────────────────────────────
  ATTACK 01  —  Excessive Data Exposure  (~2 min)
───────────────────────────────────────────────────

  [ TYPE: "Retrieve my vehicle ID and VIN." ]

I don't click anything. I just ask.

  [ TYPE: "Fetch my dashboard data." ]

Watch what comes back — not just what's on screen.

My phone number is in that response.

It never gets displayed on the web page.

If it's not shown, why is the API returning it?

This is how breaches start — an API that returns
more than it should, and a developer who trusted
the front-end to hide it.


───────────────────────────────────────────────────
  ATTACK 02  —  BOLA  (~2 min)
───────────────────────────────────────────────────

  [ TYPE: "Can you find other users' vehicle IDs?" ]

These are UUIDs — cryptic, random, not guessable.

The developers did the right thing there.

But Claude didn't need to guess.

It found them leaking through the community forum API.

Attack 01's overexposed data handed us the keys
to attack 02.


───────────────────────────────────────────────────
  ATTACK 03  —  Mass Enumeration  (~2 min)
───────────────────────────────────────────────────

  [ TYPE: "Get the vehicle location for each of those IDs." ]

That's the gap.

Authentication tells you who I am.
Authorization tells you what I'm allowed to see.

This API checked the first and skipped the second.

  [ TYPE: "Plot those locations on a map." ]

  [ Let the map render. ]

I'm logged in as a legitimate user. I can see my own car.

But I now have the location of every vehicle
in the system.

A car thief with a valid account just got a map.

  DEMO LANDING  —  ~1 min
═══════════════════════════════════════════════════

That's the threat model.

Not a sophisticated attacker at a terminal —
a language model issuing API calls, following
breadcrumbs, finding flaws in seconds.

And blocking AI agents doesn't fix it.

They call APIs the same way your own applications do —
standard HTTP, valid tokens, normal request patterns.

There is no AI fingerprint.

If you somehow blocked every agent, a human with
Chrome DevTools finds the same flaws.
You haven't fixed anything.

More importantly: AI agents are the future your
customers are already adopting.

You can't opt out. You can only be ready.

Fix the foundation. Three questions —
every endpoint, every request:

  Visibility:      Do we know every API exists?

  Validation:      Are we returning only what
                   the client needs?

  Authorization:   Does this token own this resource?

Answer yes to all three — and the attacks you just
saw don't work. Not slower. They stop.
